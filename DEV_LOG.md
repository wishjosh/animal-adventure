### 레벨 3 들개 격리 및 모바일 HUD 수정

**문제**
- 들개 무리를 관목/울타리로 격리해도 미션이 통과되지 않는 문제가 있었다.
- 들개가 관목이나 울타리 밖으로 나가려 할 때 몸 일부가 블록과 겹쳐 보였다.
- 모바일 화면에서 방위표 아래 목적지 표시 박스가 우측 조작 버튼과 겹쳤다.

**수정**
- `js/world.js`에 레벨 3 격리벽 판정 헬퍼를 추가해 지면 위에 설치된 관목, 울타리, 돌, 흙 등을 정확히 감지하도록 했다.
- `js/Level3Logic.js`의 들개 격리 BFS가 기존처럼 벽 위쪽 빈 칸을 검사하지 않고 실제 설치된 격리벽을 검사하도록 변경했다.
- 들개 이동 시 중심점만 검사하던 방식을 몸 반경 샘플 검사로 바꿔 관목/울타리에 파고들기 전에 방향을 틀도록 했다.
- 모바일/작은 화면에서는 방위표와 목적지 박스를 왼쪽 상단으로 배치해 우측 `빠름/점프/하강` 버튼과 겹치지 않게 했다.

### 레벨 3 먹이사슬 퍼즐 모바일 드래그 수정

**문제**
- 레벨 3 후반 먹이사슬 퍼즐 타워에서 카드를 슬롯으로 드래그할 수 없었다.
- 기존 구현은 HTML5 `dragstart/drop` 이벤트에 의존했는데, iPhone/iPad 터치 환경에서는 이 방식이 안정적으로 동작하지 않는다.

**수정**
- `js/Level3Logic.js`의 먹이사슬 퍼즐 카드 배치 로직을 `placeCardInSlot()`로 분리했다.
- 기존 마우스 HTML5 드래그를 유지하면서, 모바일용 `PointerEvent` 기반 드래그를 추가했다.
- 작은 화면에서 더 쉽게 조작할 수 있도록 `카드 탭 -> 슬롯 탭` 배치 방식도 추가했다.
- 배치한 카드는 PC에서는 더블클릭, 모바일에서는 탭으로 다시 제거할 수 있게 했다.

**검증**
- `Get-ChildItem -Recurse -Filter *.js | ForEach-Object { node --check $_.FullName }`
- `git diff --check`

## 2026-06-13

### 모바일 시점 이동 터치가 중간에 멈추는 문제

**문제**
- 게임 중 모바일 시점 이동 터치가 갑자기 먹히지 않는 상황이 있었다.
- 원인은 `lookPointerId`가 `pointerup`, `pointercancel`, 포인터 캡처 손실, 브라우저 포커스 전환 등에서 정상 초기화되지 않으면 이후 터치가 새 시점 조작으로 인식되지 않는 구조였다.

**수정**
- `js/main.js`에 터치 시점 상태를 강제로 정리하는 흐름을 추가했다.
- `lostpointercapture`, 모달 표시/해제, `blur`, `pagehide`, `visibilitychange` 같은 중단 상황에서도 시점 입력 상태가 풀리도록 했다.
- 걷기와 비행 이동값도 함께 조정했다.

**커밋**
- `77d924e Fix mobile touch look reset and movement speed`

### 빠른 이동 중 게임이 튕기고 처음으로 돌아가는 문제

**문제**
- 이동 속도 증가 후 빠르게 이동하거나 비행할 때 모바일 브라우저에서 메모리 압박이 커져 게임이 새로 시작되는 현상이 의심됐다.

### 레벨 4 홍수 대응 안내 개선 진행

**문제**
- 레벨 4 마지막 홍수 단계에서 설명은 길지만 실제로 어디를 막아야 하는지, `수달 보내기` 버튼이 어디 있는지, 둑 보강 위치가 어딘지 알기 어려웠다.
- 기존 수달 스킬은 수달이 합류하지 않은 상태에서는 안내를 거부했고, 가이드 큐브도 작고 짧게 보여 발견하기 어려웠다.

**작업 중인 수정**
- `js/Level4Logic.js`에 홍수 취약 둑 좌표를 `getFloodWeakPoints()`로 모았다.
- 둑이 비어 있거나 물이면 `isLeveeOpen()`으로 보강 필요 상태를 판정하도록 했다.
- `showLeveeGuides()`와 `clearLeveeGuides()`를 추가해 보강 지점을 큰 기둥, 바닥 표시, 아이콘, 목적지 나침반으로 보여주도록 했다.
- 홍수로 둑이 유실될 때 바로 해당 위치를 강조하도록 했다.
- 레벨 4에서는 수달 버튼을 눌렀을 때 보호자 상태가 아직 동기화되지 않았더라도 안내가 나오도록 보완했다.
- `getVisibleMissionY()`가 로드된 블록 높이뿐 아니라 절차 지형 높이도 참고하게 해 시멘트 보와 쓰레기 오브젝트가 지형/수면 위로 더 안정적으로 배치되도록 조정했다.
- 시멘트 보 메시가 더 눈에 띄도록 형태와 라벨을 개선했고, 미션 오브젝트 배치 헬퍼 `placeLevel4MissionMesh()`를 추가했다.

**검증**
- `node --check js/Level4Logic.js`
- `for f in js/*.js; do node --check "$f" || exit 1; done`

**상태**
- 홍수 안내 및 레벨 4 배치 개선은 이후 커밋/푸시되었다.
- 관련 커밋: `83d8ffb`, `03c05fe`

### 레벨 4 지형과 미션 배치의 구조적 불일치

**확인한 점**
- 현재 배경 지형은 `js/world.js`의 절차 생성 함수가 생물군계 가중치, 노이즈, 물 높이, 강 깎임 값을 섞어 만든다.
- 반면 레벨 4 미션 오브젝트는 `js/Level4Logic.js` 안의 고정 좌표에 많이 의존한다.
- 예: 소야 시작점, 펜션 예정지, 시멘트 보 좌표, 홍수 취약 둑 좌표, 산비탈 버드나무 조건 등이 실제 지형의 강/평지/구릉과 강하게 결합되어 있지 않다.

**결론**
- 사용자가 본 것처럼 시멘트 보나 둑 보강 위치가 강의 맥락과 어긋나 보일 수 있다.
- 근본 해결은 레벨 4 전용 미션 지형을 명시적으로 만들거나, 절차 지형에서 실제 강둑/구릉/평지를 탐색해 미션 좌표를 동적으로 정하는 방식이 필요하다.
